# Firestore Setup Guide for Task System

## Prerequisites
- Firebase project created
- Firestore database enabled
- Environment variables configured in `.env.local`

## Firestore Security Rules

To ensure user data security, add these security rules to your Firestore:

1. Go to Firebase Console → Firestore Database → Rules
2. Replace the existing rules with:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Tasks collection - users can only read/write their own tasks
    match /tasks/{taskId} {
      allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && request.auth.uid == request.resource.data.userId;
    }
  }
}
```

## Firestore Indexes

For optimal query performance, create these composite indexes:

1. Go to Firebase Console → Firestore Database → Indexes
2. Create the following composite indexes:

### Index 1: Tasks by user and due date
- Collection: `tasks`
- Fields:
  - `userId` (Ascending)
  - `dueDate` (Ascending)
- Query scope: Collection

### Index 2: Tasks by user, completion status, and due date
- Collection: `tasks`
- Fields:
  - `userId` (Ascending)
  - `completed` (Ascending)
  - `dueDate` (Ascending)
- Query scope: Collection

**Note**: You can also create these indexes automatically by running your app. When you get an error with an index creation link, click it to auto-create the index.

## Testing the Task System

### Option 1: Use the TaskManager Component (Temporary)

Add the TaskManager component to your dashboard for testing:

1. Open `app/dashboard/page.tsx`
2. Import the component:
```typescript
import TaskManager from '@/components/TaskManager';
```

3. Add it temporarily to the dashboard (e.g., after the Today's Goal card):
```tsx
{userId && <TaskManager userId={userId} onTaskAdded={() => window.location.reload()} />}
```

4. Use the "Add Sample Tasks" button to populate your dashboard with test data
5. Remove the component once testing is complete

### Option 2: Add Tasks Programmatically

Create a test script or use the browser console:

```javascript
import { addTask } from '@/lib/tasks';

const userId = 'your-user-id'; // Get from auth

await addTask({
  title: 'Complete Math Assignment',
  subject: 'Mathematics',
  dueDate: new Date(),
  userId: userId,
});
```

### Option 3: Use Firestore Console

1. Go to Firebase Console → Firestore Database → Data
2. Create a new collection called `tasks`
3. Add documents with the following fields:
   - `title` (string): "Complete Assignment"
   - `subject` (string): "Mathematics"
   - `dueDate` (timestamp): Select today's date
   - `completed` (boolean): false
   - `createdAt` (timestamp): Select current time
   - `userId` (string): Your user ID from Firebase Auth

## Verifying the Setup

1. **Authentication**: Ensure user is logged in
2. **Dashboard displays**: Check that task counts update
3. **Today's Goal card**: Shows tasks due today with correct count
4. **Progress percentage**: Calculated correctly based on completed tasks
5. **Upcoming Tasks card**: Displays incomplete tasks ordered by due date
6. **Task toggle**: Click checkboxes to mark tasks complete
7. **Empty state**: If no tasks exist, see the "No upcoming tasks" message

## Troubleshooting

### No tasks appearing
- Check Firebase Console for data in the `tasks` collection
- Verify the `userId` matches your authenticated user ID
- Check browser console for errors
- Ensure Firestore security rules are properly configured

### Permission denied errors
- Verify security rules allow read/write for authenticated users
- Check that `userId` field in tasks matches the authenticated user's ID
- Ensure user is properly authenticated

### Query errors (missing index)
- Click the error link to auto-create the required index
- Wait 1-2 minutes for index to build
- Refresh the page

## Task Data Structure

Each task document in Firestore contains:

```typescript
{
  id: string;              // Auto-generated by Firestore
  title: string;           // Task title
  subject: string;         // Subject/category
  dueDate: Timestamp;      // Due date
  completed: boolean;      // Completion status
  createdAt: Timestamp;    // Creation timestamp
  userId: string;          // Owner user ID
}
```

## API Reference

### Functions in `lib/tasks.ts`:
- `addTask(taskInput)` - Create a new task
- `updateTask(taskId, updates)` - Update existing task
- `deleteTask(taskId)` - Delete a task
- `toggleTaskCompletion(taskId, currentStatus)` - Toggle completion
- `getUserTasks(userId)` - Get all user tasks
- `getTasksDueToday(userId)` - Get tasks due today
- `getUpcomingTasks(userId, limit)` - Get upcoming incomplete tasks

### Custom Hook `useTasks(userId)`:
Returns:
- `tasks` - All user tasks
- `tasksDueToday` - Tasks due today
- `upcomingTasks` - Upcoming incomplete tasks
- `loading` - Loading state
- `error` - Error message
- `fetchTasks()` - Manually refresh tasks
- `toggleTaskCompletion(taskId, currentStatus)` - Toggle task
- `getTaskStats()` - Get today's statistics
