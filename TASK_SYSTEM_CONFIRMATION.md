## How It Works Across Pages

### Dashboard (`app/dashboard/page.tsx`)

**Hook Used:** `useTasks(userId)`
- **File:** `hooks/useTasks.ts`
- **Fetch Method:** Manual queries with `getDocs`
- **Updates:** Manual refresh with `fetchTasks()`

**Functions:**
```typescript
const {
  tasksDueToday,        // Tasks due today
  upcomingTasks,        // Upcoming tasks (limit 5)
  loading,              // Loading state
  error,                // Error state
  toggleTaskCompletion, // Toggle completion
  getTaskStats,         // Get stats (completed/total/%)
  fetchTasks,           // Manual refresh
} = useTasks(userId);
```

**Queries:**
1. `getTasksDueToday(userId)` - Tasks due today only
2. `getUpcomingTasks(userId, 5)` - Next 5 incomplete tasks

---

### Planner (`app/planner/page.tsx`)

**Hook Used:** `useTasksRealtime(userId)`
- **File:** `hooks/useTasksRealtime.ts`
- **Fetch Method:** Real-time listener with `onSnapshot`
- **Updates:** Automatic via Firestore listener

**Functions:**
```typescript
const {
  tasks,           // All user tasks (real-time)
  loading,         // Loading state
  error,           // Error state
  createTask,      // Add new task
  editTask,        // Update task
  removeTask,      // Delete task
  toggleComplete,  // Toggle completion
} = useTasksRealtime(userId);
```

**Features:**
- Real-time updates (no manual refresh needed)
- CRUD operations (Create, Read, Update, Delete)
- Automatic UI synchronization

## Data Flow

### Creating a Task (Planner)

```
User fills TaskForm in Planner
    ↓
createTask(taskInput) called
    ↓
addTask() writes to Firestore
    ↓
Real-time listener detects change
    ↓
Planner UI updates automatically
    ↓
Dashboard queries fetch new task (on next visit/refresh)
    ↓
Dashboard shows new task in "Today's Focus" or "Upcoming"
```

### Completing a Task (Dashboard)

```
User checks checkbox on Dashboard
    ↓
handleTaskToggle(taskId, completed) called
    ↓
toggleTaskCompletion() updates Firestore
    ↓
Dashboard completion animation plays (2s)
    ↓
fetchTasks() refreshes dashboard data
    ↓
Dashboard UI updates with new completion status
    ↓
Planner real-time listener detects change
    ↓
Planner UI updates automatically
```

### Completing a Task (Planner)

```
User checks checkbox on Planner
    ↓
toggleComplete(taskId, completed) called
    ↓
toggleTaskCompletion() updates Firestore
    ↓
Real-time listener detects change
    ↓
Planner UI updates automatically (instant)
    ↓
Dashboard shows updated status on next visit
```

## Security

### Firestore Rules

Tasks are user-specific:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tasks/{taskId} {
      // Users can only read/write their own tasks
      allow read, write: if request.auth != null 
        && request.auth.uid == resource.data.userId;
    }
  }
}
```

### Benefits:
- ✅ Users can't see others' tasks
- ✅ Users can't modify others' tasks
- ✅ Authentication required
- ✅ Firebase Auth integration
